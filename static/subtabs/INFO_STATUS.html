Dashboard Status

System Status

The dashboard is currently fully operational.
All data feeds, charts, explorer sections, and metric views are running reliably and without known issues. No problems have been detected so far.

Release Status

The dashboard is currently in its initial release phase. All core features, data feeds, and visualizations are stably implemented and ready for productive use.
Additional features, refinements, and optimizations will be introduced gradually in future updates.

Responsive support for mobile devices and tablets is still in an early stage of development.
Basic readability and usability are provided, but certain layouts and tables are still being unified and optimized.

For the best overall overview and user experience, it is currently recommended to use the dashboard primarily in desktop view.

Planned Features

The Indicators section is still under active development. At this time, no specific release date can be provided for the first finalized indicator.
During the initial development phase, the primary focus has been on ensuring that existing tabs and core features are fully functional, performant, and stable before expanding into additional areas.

Some visualizations — particularly in the Network, Nodes, and Miner sections — are currently simplified by design and serve primarily for high-level conceptual orientation.
Further technical depth and extended detail will be added incrementally.

In addition, a significant expansion of existing chart visualizations and their underlying historical data sets is planned.
Specific scope, content, and timelines cannot yet be defined at this stage.

Planned System Updates

System updates, technical optimizations, and structural adjustments are generally performed outside of peak usage hours.
Timing is chosen to minimize any potential impact on ongoing traffic and usage.

The goal is to apply all updates during live operation. Only in exceptional cases may it become necessary to temporarily restrict certain features or take the dashboard offline for a short period.
Scheduled system updates will be announced at least 24 hours in advance within the status section.
